## Repository rules

### Specs and plans

- Plans/specs must be created in ./specs.
- Spec filenames must be prefixed with YYMMDD-HHMM-<name>.md.
- Before creating a plan, ask any clarifying questions.
- Maintain specs/INDEX.md.
- Before reading or adding specs, check specs/INDEX.md.
- When adding a spec, add an entry to specs/INDEX.md in the same change.
